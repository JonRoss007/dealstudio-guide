NDA Email Automation Flow

Up to 4 automated touchpoints with a buyer before the deal is marked lost. All fully automated — no broker action required.

⚠️
Requirements for NDA emails to send: Seller Opportunity must be in Seller Pipeline → On Market stage → Status: Open. Internal ID and Public Advertised Title must be completed.
1
NDA Email 1
⚡ Immediate
Sent as soon as a buyer inquiry is received. Introduces the listing and sends a link to the electronic Confidentiality / NDA form.
📍 Buyer enters Inquiry Received stage on Buyer Pipeline
2
NDA Follow-Up 1
⏱ +3 Days
Sent if buyer has not yet signed the NDA. Gentle reminder with NDA link included.
📍 Only sends if NDA still unsigned at Day 3
3
NDA Follow-Up 2
⏱ +7 Days
Second follow-up if buyer has still not responded. Sent 7 days after the first follow-up.
📍 Only sends if NDA still unsigned at Day 10 total
4
NDA Final
⏱ +10 Days
Final outreach. If no response after this email, the deal is automatically marked as lost 3 days later.
⛔ Deal marked Lost if no response within 3 days
After NDA is signed
1
NDA Automatically Signed & Stage Moves
Buyer completes the NDA electronically. The system automatically moves the buyer from Inquiry Received → NDA Signed. Buyer receives a copy of the signed NDA by email.
2
Executive Summary / Blind Profile Email (15 minutes after)
If a Blind Profile URL is present in the Seller Opportunity (Buyer Automations Tab or Opportunity Details), an email with the link is auto-sent 15 minutes after NDA sign. ⚠ If field is blank — email will NOT send. Edit via Marketing → Snippets → DS_2.0.0_NDA Follow Up
3
Broker Qualifies Buyer
Broker reviews the buyer, checks biometrics (if using DealBuilder), and decides whether to proceed to sending the CIM/CBR. No automation — this is a manual broker decision.
4
CIM / CBR Access (Broker Action Required)
Via DealBuilder (recommended): Go to buyer → Actions → Give Presentation Access → moves buyer to CIM/CBR Sent stage automatically + gives data room access.

Via Pipeline (legacy): Drag buyer to CIM/CBR Sent stage → select Yes to send CBR email. This only sends the CBR email and CIM access — does NOT give data room access.
⚠ Data room access must be granted through DealBuilder only

Updated Buyer workflow in DealBuilder
1
Add Buyer to DealBuilder Listing
Navigate to DealBuilder → specific listing → Add Buyer. Leave buyer in Inquiry Received stage. The NDA automation triggers automatically — no separate URL or extra steps needed.
2
Buyer Signs NDA → Exec Summary Sends
System auto-moves buyer to NDA Signed. Executive Summary/Blind Profile email sends 15 minutes later (if URL field is populated). Buyer is now visible in DealBuilder with their linked listings.
3
Broker Checks Biometrics Before Qualifying
In DealBuilder, view buyer statistics: page views on CIM, time spent per page, login count, unique views. Use this data to prioritise which buyers are genuinely interested before scheduling calls or granting CIM access.
4
Give CIM + Data Room Access
In DealBuilder: buyer → Actions → Give Presentation Access. This: (a) moves buyer to CIM/CBR Sent stage automatically, (b) grants CIM access, (c) optionally grants data room access. ⚠ Do NOT allow downloads until Due Diligence — downloads break biometric tracking
5
Buyer Accesses Buyer Portal
Buyer receives unique portal link. Login uses email + verification code only — no password or registration needed. All activity (pages viewed, time spent, documents accessed) is tracked in real time in your DealBuilder dashboard.
6
Due Diligence → Enable Downloads
When buyer moves to Due Diligence, you can then allow document downloads from the data room through DealBuilder. The system logs what was downloaded and when. Full data room access (5 levels) managed by broker — access can be revoked at any time.

AI CIMBuilder

Built directly into DealBuilder. Reduces CIM creation time by ~70% using your seller intake questionnaire, data room files, and the client's website.

✅
Status: Live and stable. Requires an OpenAI API key connected to your account. Typical usage cost: ~$3–4 per CIM generated. Output is fully editable (like PowerPoint slides) and can be exported as PDF.
What CIMBuilder uses to create your CIM
📋 Seller Intake Questionnaire
Send the questionnaire from DealBuilder. Seller completes it online via their portal. All answers are pulled into CIMBuilder automatically. The more complete the questionnaire, the better the CIM output.
📁 Data Room Files
Upload financial documents, operational information, and supporting files to the data room. CIMBuilder references the files you select when generating the CIM. Documents uploaded by seller via questionnaire go to Seller Only level automatically.
🌐 Client Website
If the client has a website, add the URL in DealBuilder. CIMBuilder will pull in relevant business description, services, and branding information to supplement the questionnaire data.
CIMBuilder workflow
1
Send Seller Intake Questionnaire
From DealBuilder, send the seller intake questionnaire to your client. Multiple questionnaire types are supported (intake, document upload, teaser/blind profile). Seller completes online — no need to manually enter their responses.
2
Upload Documents to Data Room
Files uploaded by the seller via the questionnaire go directly to the Seller Only section of the data room. You can also manually upload additional documents. Select which files CIMBuilder should use.
3
Generate CIM in CIMBuilder Tab
Go to the CIMBuilder tab within DealBuilder. The AI generates a ~17-page CIM using your selected data. The prompt structure is pre-built — review and edit the output as needed. Editing is similar to working in PowerPoint.
4
Review, Edit & Export
Review the generated CIM, make edits and adjustments. Export as PDF when ready. You can also embed listing-specific broker videos directly in the CIM slides for a richer buyer experience.
5
Upload to DealBuilder (Existing Listings)
For existing listings where you already have a CIM as a PDF — upload it directly to DealBuilder. You immediately get biometric tracking benefits (page views, time per page) even without using the AI CIMBuilder to create it.

Do's & Don'ts
🚫 Never Do This
  • ✕Delete any stages in the Buyer or Seller Pipeline
  • ✕Rename any automation-connected stage
  • ✕Delete or rename your NDA document in the Payments Tab
  • ✕Allow downloads from the data room before Due Diligence stage (breaks biometric tracking)
  • ✕Leave the Blind Profile URL blank if you want the Exec Summary email to send
  • ✕Leave the CBR URL blank if you want the legacy CBR email to send
  • ✕Try to give data room access from the pipeline — only works through DealBuilder
✅ You Can Do This
  • ✓Add new stages in between existing automation stages
  • ✓Reorder stages (as long as automation-connected ones are not moved)
  • ✓Set pipeline permissions per user (view / edit / no access)
  • ✓Embed videos in email templates (NDA, Exec Summary, etc.)
  • ✓Use tags on listings to auto-tag buyers when they inquire
  • ✓Upload existing PDF CIMs to DealBuilder to get biometric tracking
  • ✓Merge duplicate contacts (keep separate buyer opportunities)

1
Opportunity Details Tab
Confirm Pipeline: Sellers and Stage: Post Closing. Drag & drop the opportunity to set Status: Won.
2
Listing Public Data Tab
Set Current Status: Sold. Set Show on Web: Yes. Both fields must be set for the tombstone to appear on your website.

Sold Email Triggers
